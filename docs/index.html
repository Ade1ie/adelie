Workspace Management

adelie init вҖ” Initialize workspace

Creates .adelie/ with KB structure, .env config, and OS-specific context.

adelie init                          # Current directory
adelie init /path/to/project         # Specific directory
adelie init --force                  # Reinitialize (keeps .env)
Option Description
[directory] Target directory (default: .)
--force Overwrite existing .adelie/

вңЁ Auto-detects project type: Node.js, React, Vue, Next.js, Python, Rust, Go, Java (Maven/Gradle), Ruby, PHP, and sets appropriate phase.

adelie ws вҖ” Manage registered workspaces

adelie ws                            # List all workspaces
adelie ws remove 2                   # Remove workspace #2
рҹҡҖ

AI Loop Execution

adelie run вҖ” Start the autonomous loop

adelie run                                 # Auto-generates goal from specs
adelie run --goal "Build a chat app"       # Explicit goal
adelie run --once                          # Run single cycle then exit
adelie run ws 1                            # Resume workspace #1
adelie run ws 1 --goal "New direction"     # Resume with new goal
Option Description
--goal "text" High-level goal for all AI agents
--once Run exactly one cycle then exit
ws <N> Resume registered workspace #N

вҡ пёҸ If no goal is set and no spec files exist, Adelie uses a generic improvement goal. Use adelie goal set "..." or place spec files in .adelie/specs/ for better results.

Knowledge Base

The KB is a structured Markdown store in .adelie/workspace/ shared by all agents. It has 5 categories:

рҹ“җ

logic/

Architecture, project goal, design decisions.

рҹ”§

skills/

How-to guides, patterns, techniques.

рҹ“Ұ

dependencies/

Library docs, API references, changelogs.

рҹҗӣ

errors/

Known issues, error patterns, fixes.

рҹ”‘

maintenance/

Setup, deployment, infra knowledge.

adelie kb вҖ” KB management

adelie kb                   # Show file counts by category
adelie kb --clear-errors    # Delete all files in errors/
adelie kb --reset           # Wipe entire KB (requires confirmation)

adelie spec вҖ” Load specification files

Load business requirements, design docs, or API specs into KB. Supports .md, .pdf, .docx. 

adelie spec load requirements.md              # Markdown spec
adelie spec load architecture.pdf             # PDF (auto-converts)
adelie spec load api_spec.docx --category dependencies
adelie spec list                              # Show loaded specs
adelie spec remove spec_requirements          # Remove by name

adelie scan вҖ” Analyze existing codebase

adelie scan                          # Scan current directory
adelie scan --directory /path/to/src # Specific directory

adelie goal вҖ” Project goal

adelie goal                                  # Show current goal
adelie goal set "Build a realtime chat app"  # Set/overwrite goal

adelie feedback вҖ” Send feedback to running loop

adelie feedback "Focus on auth first"
adelie feedback "Stop production deploy" --priority critical
adelie feedback --list              # List pending feedback
Priority When to use
low Nice-to-have suggestions
normal Standard guidance (default)
high Important direction changes
critical Stop / emergency override

adelie research вҖ” Web research вҶ’ KB

adelie research "FastAPI WebSocket tutorial"
adelie research "Redis pub/sub" --context "high-perf" --category skills
adelie research --list              # List recent results

Changelog

0.3.3 2026-04-10 Fix в–ҫ

рҹҗӣ Fixed

  • Review Failure Logging вҖ” Rejected code details now logged to KB errors, preventing Expert AI from re-issuing identical failing tasks.
  • Expert AI Context Gap вҖ” last_coder_result now included in system state for cycle-to-cycle continuity.
  • Reviewer Staging Path вҖ” Reviewer now reads from STAGING_ROOT instead of PROJECT_ROOT.
  • Scaffolding Warning Severity вҖ” Reduced from вҡ пёҸ CRITICAL to в„№пёҸ NOTE to prevent model over-reaction.
  • Framework-Specific Coder Guidelines вҖ” Next.js 'use client' rules, correct package names, etc. injected into Coder prompt.
0.3.2 2026-04-10 Feature в–ҫ

вңЁ Added

  • Configurable LLM Timeout вҖ” New LLM_TIMEOUT env var (default: 300s, up from hardcoded 120s). Prevents timeouts on large models like gemma4:31b via Ollama Cloud.
0.3.1 2026-04-10 Fix в–ҫ

рҹҗӣ Fixed

  • Framework Detection вҖ” Scaffolding check no longer assumes all JS/TS projects use Vite. Added _detect_framework() supporting Next.js, Nuxt, Remix, SvelteKit, and Angular with framework-appropriate entry file checks.
  • Path Safety вҖ” Fixed ".." in filepath substring match incorrectly blocking Next.js catch-all routes like [...nextauth]. Now uses Path.parts for segment-level traversal detection.
  • Coder Context вҖ” Auto-included config files for coders now cover Next.js, Nuxt, SvelteKit, Remix, and Angular configs alongside Vite.
0.3.0 2026-04-09 Major в–ҫ

Transforms Adelie from a static pipeline executor into a self-configuring, policy-enforced, production-aware AI harness. Consolidates v0.2.16вҖ“v0.2.20.

рҹ”§ Meta Harness вҖ” Dynamic Pipeline

  • HarnessManager вҖ” Dynamic JSON state machine. Expert AI reconfigures pipeline at runtime via MODIFY_HARNESS.
  • DynamicAgent вҖ” Runtime-created agents with 3-tier permissions. Snapshot rollback on failure.

рҹӣЎпёҸ Policy Engine вҖ” Declarative Constraints

  • PolicyEngine вҖ” Enforces constraints.yaml rules (pattern/ast/file). PolicyGate blocks promotion on violation.
  • AST Checker вҖ” Detects eval(), exec(), wildcard imports, missing docstrings.

рҹ§  Memory Harness вҖ” Selective Forgetting

  • Phase Scope Filter вҖ” KB files visible only during designated phases.
  • Archive Manager вҖ” Auto-archives resolved errors + completed-phase docs. Summary Tree preserves awareness.

рҹ“Ў Production Bridge вҖ” CI/CD Feedback

  • 3 Adapters вҖ” GitHub Actions, Sentry, Custom MCP. Critical verdict вҶ’ ERROR + hotfix.
  • Graceful Degradation вҖ” Missing tokens = adapter silently disabled.

вӣ” Human Intercept & Monitoring

  • CLI вҖ” /intercept, /policy, /health, /memory, /harness, enhanced /status.
  • Dashboard вҖ” Intercept button, Production Health, Policy Engine, Memory Harness, Pipeline Visualizer panels.

  • Archive Manager вҖ” Stale errors (3+ cycles) and completed-phase docs auto-archived to archive/, removed from active queries.

0.2.17 2026-04-09 Feature в–ҫ

вңЁ Added

  • PolicyGate вҖ” Deterministic gate between Reviewer AI and staging promotion. Blocks violating code with automatic Coder retry loop.
